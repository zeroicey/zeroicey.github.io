OpenCLAW 原理解析

发表于 2026-04-06 分类于 技术解析 本文字数： 3.5k 阅读时长 ≈ 3 分钟

OpenCLAW（原名 Clawdbot、Moltbot）是一个自托管、本地优先的 AI 助手平台，支持 20+ 消息通道和 AI 模型提供商。

OpenCLAW 原理解析

更新日期：2026-04-06

什么是 OpenCLAW？

OpenCLAW（原名 Clawdbot、Moltbot）是一个自托管、本地优先的 AI 助手平台。

核心理念：

  • 你在自己的硬件上运行一个始终在线的进程叫 Gateway
  • Gateway 连接到消息应用（WhatsApp、Telegram、Discord、iMessage、Slack 等）
  • 接收消息 → 运行 Agent（大脑）→ 可选调用工具 → 发送响应

GitHub: 241,000+ stars，20+ AI 模型提供商

核心架构

Gateway = Agent 的头脑，Node = Agent 的手足

重要澄清：

  • Gateway：是 Agent 的大脑，负责推理、HEARTBEAT 心跳机制
  • Node：是 Agent 的手足，负责工具执行、输入输出

五大核心组件

1. ChannelPlugin（通道插件）

统一接口适配不同消息平台：

支持 20+ 平台：WhatsApp, Telegram, Discord, Slack, iMessage, Signal, Line, etc.

2. Gateway（网关）

  • Session Management：管理每个用户的会话状态
  • Message Routing：消息路由和分发
  • Connection Management：WebSocket 长连接管理

3. Agent Runtime（智能体运行时）

  • Pi Agent Core：核心智能体逻辑
  • Message Processing Loop：消息处理循环
  • State Machine：状态机管理

4. LLM Providers（模型层）

支持 20+ 模型提供商：

  • OpenAI (GPT-4o, o1, etc.)
  • Anthropic (Claude)
  • Google Gemini
  • Ollama (本地模型)
  • Azure OpenAI
  • 等等

5. Tool Execution & Sandboxing（工具执行与沙盒）

  • Sandbox Isolation：隔离执行，确保安全
  • Skill Extensions：技能扩展机制
  • Permission Control：细粒度权限控制

三大多智能体协调机制

1. SubAgent（子智能体）

主 Agent 派生的子任务执行单元：

  • 共享主 Agent 上下文
  • 独立决策
  • 任务完成后回归

2. Agent Teams（智能体团队）

多个 Agent 协同工作：

  • 角色分配
  • 任务协调
  • 资源共享

3. AgentToAgent（跨 Agent 通信）

不同 Agent 之间直接通信：

  • 消息传递
  • 状态同步
  • 协作决策

Session & Memory Management（会话与内存管理）

OpenCLAW 的持久化机制：

  1. Session State：每个对话的会话状态
  2. Memory：长期记忆存储
  3. Context Window：上下文窗口管理
  4. State Persistence：状态持久化

Skills and Extensions（技能与扩展）

OpenCLAW 的可扩展性来源：

  • Built-in Skills：内置技能
  • Custom Skills：自定义技能
  • Plugin System：插件系统
  • Tool Registry：工具注册表

Tool Execution & Sandboxing（工具执行与沙盒）

安全模型

OpenCLAW 将网关主机视为信任边界：

  • 传入消息被视为不可信输入
  • 默认拒绝执行危险操作
  • 沙盒提供隔离执行环境

权限控制

  • 细粒度的工具调用权限
  • 操作审计日志
  • 可配置的信任策略

HEARTBEAT 机制

Agent 的心跳机制：

  • 定期检查 Agent 状态
  • 维持长连接
  • 超时自动恢复
  • 状态同步

与 Claude Code 的关系

OpenCLAW 利用了 Claude Code 的 Hooks 回调机制 + Agent Teams 特性：

“其解決策是非常優雅な——結合了 Claude Code 的 Hooks 回調機制和最新的 Agent Teams 多智能體協作功能”

OpenCLAW vs ChatGPT

特性 ChatGPT OpenCLAW
模式 “回答问题” “执行操作”
部署 云端 本地/自托管
能力 文本对话 shell命令、文件读写、浏览器控制
自主性 低（需要人工确认） 高（自主执行）
隐私 数据上传云端 数据留在本地
架构 单一模型 多层架构 + 多 Agent
